skupper.core.skupper_option role – Provides common parameters to be used across the skupper.core roles.

Note

This role is part of the skupper.core collection (version 1.6.0).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it use: ansible-galaxy collection install skupper.core.

To use it in a playbook, specify: skupper.core.skupper_option.

Entry point main – Provides common parameters to be used across the skupper.core roles.

New in skupper.core 1.6.0

Synopsis

  • Provides common parameters to be used across the skupper.core roles (except skupper_cli_install). This role is not supposed to be executed directly, as the intent is simply to be used as a dependency to all other skupper.core roles that need to use the global options.

Parameters

Parameter

Comments

skupper_option_kubeconfig

string

added in skupper.core 1.6.0

Kubeconfig file to use, if empty uses the default under ${HOME}/.kube/config

Default: ""

skupper_option_kubecontext

string

added in skupper.core 1.6.0

skupper_option_namespace

string

added in skupper.core 1.6.0

Default: ""

string

Target platform (kubernetes or podman)
