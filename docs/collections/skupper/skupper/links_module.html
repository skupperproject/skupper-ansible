skupper.skupper.links module – Update links based on provided links list

Note

This module is part of the skupper.skupper collection (version 1.8.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install skupper.skupper. You need further requirements to be able to use this module, see Requirements for details.

To use it in a playbook, specify: skupper.skupper.links.

New in skupper.skupper 1.1.0

  • Synopsis

  • Requirements

  • Parameters

  • Examples

Synopsis

  • Updates the links defined by the respective site, based on the links list defined through the host variables.

  • Existing links that cannot be mapped to an inventory host will be deleted.

  • Links with a corresponding token (provided by skupper_token role) or manually through the token property will be created.

  • If a corresponding token cannot be found for a requested link, it will fail.

Note

This module has a corresponding action plugin.

Requirements

The below requirements are needed on the host that executes this module.

  • python >= 3.9

  • kubectl if using kubernetes platform

  • podman v4+ if using podman as the site platform

Parameters

Parameter

Comments

context

string

KUBECONFIG context to use

create

list / elements=dictionary

List of links to create for a given site

This list is automatically provided by the links plugin based on host variable named links

More information on how to define a link can be found at the skupper_link role documentation

For links to be created the skupper_token role must have been invoked for the target host

cost

integer

Cost of the link

host

string

The ansible_inventory hostname that the respective link is intending to use as a target

name

string

Name of the link to be created

token

string

Token to use in case skupper_token role hasn’t been called (generated token is not available)

delete

list / elements=dictionary

List of link names to be deleted

Existing links must have been loaded earlier by calling skupper.skupper.links_load module

name

string / required

Name of the link to be deleted

hostname

string

Ansible’s inventory_hostname used to represent the given Skupper site

This is automatically provided by the action plugin

kubeconfig

string

KUBECONFIG file to be used (defaults to ${HOME}/.kube/config)

namespace

string

Kubernetes namespace to run the Skupper site

platform

string

Platform to be used (kubernetes or podman)

podman_endpoint

string

Podman endpoint to use when managing a Skupper site

Examples

- name: Updating links
  skupper.skupper.links:
    create:
      - host: host-a-from-inventory
      - name: site-b
        token: |
          TOKEN IN YAML FORMAT
    delete:
      - name: site-c
