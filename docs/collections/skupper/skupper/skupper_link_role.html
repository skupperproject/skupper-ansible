skupper.skupper.skupper_link role – Maintains existing links on a site

Note

This role is part of the skupper.skupper collection (version 1.8.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it use: ansible-galaxy collection install skupper.skupper.

To use it in a playbook, specify: skupper.skupper.skupper_link.

Entry point main – Maintains existing links on a site

New in skupper.skupper 1.6.0

Synopsis

  • Maintains existing links on a site based on the provided links object.
    All desired links must be provided as part of the skupper_link_list list.
    Links must specify a target hostname (a valid host from the Ansible inventory), in such
    case the token for the respective link is expected to be available as a host fact (for the target host)
    named skupper_token_generated_token. This fact is set automatically if the skupper.skupper.skupper_token role
    is invoked prior to this one.
    Alternatively if the intention is to use a statically provided token, then the token parameter of the link object
    must be populated as well as the name of the link.
    This role uses the skupper.skupper.links_load and skupper.skupper.links modules to ensure all desired links are defined.

Parameters

Parameter

Comments

skupper_link_list

list / elements=dictionary

added in skupper.skupper 1.6.0

List of all desired links to exist at a given site. Links that are defined on the list, but does not exist, will be created. The links that exist on the Skupper site but are not defined through this list will be removed.

cost

integer

added in skupper.skupper 1.6.0

The cost of the link.

Default: 1

host

string

added in skupper.skupper 1.6.0

The Ansible Inventory hostname to use as a target for this link. It can be populated if the skupper.skupper.skupper_token role has been invoked earlier, of when using the skupper.skupper.skupper_setup role (which also invokes the skupper_token role as a dependency. In case the intention is to use a static token, this argument must not be provided.

name

string

added in skupper.skupper 1.6.0

The name of the link to be defined at the target site. When using a static token, this argument is mandatory, as it is used to determine if the link already exists.

token

string

added in skupper.skupper 1.6.0

A static token to be used when creating this link. When a static token is provided, the name argument must also be set and the host argument must not be provided. The token must be represented as a YAML string.
