skupper.skupper.skupper_service role – Maintains existing services on a site

Note

This role is part of the skupper.skupper collection (version 1.8.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it use: ansible-galaxy collection install skupper.skupper.

To use it in a playbook, specify: skupper.skupper.skupper_service.

Entry point main – Maintains existing services on a site

New in skupper.skupper 1.6.0

Synopsis

  • Maintains existing services on a site based on the provided services object.
    All desired services must be provided as part of the skupper_service_list list.
    Existing services that are no longer defined as part of the skupper_service_list list will be removed.
    Modifications done to existing service definitions will cause the service to be deleted and created.
    Targets will be maintained similarly.
    This role uses the skupper.skupper.services_load and skupper.skupper.services modules to ensure
    all desired services are defined.

Parameters

Parameter

Comments

skupper_service_list

list / elements=dictionary

added in skupper.skupper 1.6.0

List of all desired services to exist at a given site. Services that are defined on the list, but does not exist, will be created. The services that exist on the Skupper site but are not defined through this list will be removed. Changes to existing service definition will cause the service to be deleted and created. Targets will be maintained similarly.
