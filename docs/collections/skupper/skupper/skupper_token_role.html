skupper.skupper.skupper_token role – Creates a Skupper token that can be used by other sites to establish links

Note

This role is part of the skupper.skupper collection (version 1.8.3).

It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it use: ansible-galaxy collection install skupper.skupper.

To use it in a playbook, specify: skupper.skupper.skupper_token.

Entry point main – Creates a Skupper token that can be used by other sites to establish links

New in skupper.skupper 1.6.0

Synopsis

  • Creates a Skupper token that can be used by other sites to establish links.
    The token is generated as a host fact named skupper_token_generated_token.

Parameters

Parameter

Comments

skupper_token_expiry

string

added in skupper.skupper 1.6.0

Duration for which a token claim is considered as valid. This is only used when skupper_token_type=claim and skupper_option_platform=kubernetes.

Default: "15m0s"

skupper_token_ingress_host

string

added in skupper.skupper 1.6.0

The ingress-host to use in the generated token on Podman sites. Usually when a podman site has more than one ingress host, this might be needed. This can only be used when skupper_option_platform=podman. If absent or empty, Skupper will use the first available ingress host.

Default: ""

skupper_token_name

string

added in skupper.skupper 1.6.0

Name of the token that will be created. When using skupper_token_type=claim, only one token can exist with a given name.

Default: "skupper"

skupper_token_password

string

added in skupper.skupper 1.6.0

Password to be set of the token claim. If empty, a random password will be generated (default). This is only used when skupper_token_type=claim and skupper_option_platform=kubernetes.

Default: ""

skupper_token_type

string

added in skupper.skupper 1.6.0

The token type to use. When using skupper_option_platform=podman, the only allowed value is cert.

Choices:

  • "claim" ← (default)

  • "cert"

skupper_token_uses

integer

added in skupper.skupper 1.6.0

Number of times that the given token claim can be redeemed. This is only used when skupper_token_type=claim and skupper_option_platform=kubernetes.

Default: 1
